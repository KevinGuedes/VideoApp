Video App

Proof Of Concpet

The purpose of this application is to test different libraries and strategies for video rendering

Conclusion

  • The packages used (and probably all the others) will work with an URL or with the path of the video
  • The packages are wrappers for the HTML 5 video tag
  • It is possible to get the videos from back-end through different strategies
  • Whitout the video type specification it works properly (Probably getting the type under the hood)
  • Sending a webm video to a video tag with type audio/mp4 is working properly
  • Suported types are: OGG, WEBM and MP4 (OGG is audio only)
  • No support for avi files. To display this type of video it is necessary to use 'old school' techniques (Bugged and unstable)
  • Unable to prevent download with right click
  • Bugged with large videos and api endpoint in the source
  • The best: Blob. Download the video before playing it

Go To Setup

  • Allow video streaming on back end
  • Use Vime as main package because it has a good documentation
  • Only MP4 and WEBM video files are allowed
